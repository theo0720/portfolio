MQ

Entreprise d'imprimerie

Maqprint

Second stage réalisé chez Maqprint, avec des missions plus techniques et une autonomie accrue, permettant de consolider les compétences acquises lors du premier stage.

📍 Limoges, 87 🏭 Imprimerie / Communication 👥 PME — < 20 salariés 📅 5 janvier - 13 février

Missions réalisées

1

Développement d’une application de gestion de calendrier

Mise en place d’une application web permettant d’afficher un calendrier mensuel dynamique, avec navigation entre les mois et visualisation des jours. Interface permettant à l'administrateur de sélectionner une date afin d’interagir avec les événements associés.

2

Implémentation du système de gestion des événements (CRUD)

Développement des fonctionnalités permettant de créer, lire, mettre à jour et supprimer des événements dans une base de données. Chaque événement comprend un titre, une date, une plage horaire, une salle associée ainsi qu’une liste de participants.

3

Conception et gestion de la base de données

Création et structuration d’une base de données relationnelle sous MySQL comprenant plusieurs tables : événements, personnes, salles et table de liaison (event_persons). Mise en place des relations entre les entités afin de garantir la cohérence des données.

4

Développement de l’interface de l'administrateur (front-end)

Création d’une interface ergonomique et responsive en HTML et CSS permettant une visualisation claire du calendrier, une interaction intuitive avec les jours et une bonne lisibilité des événements

5

Implémentation du moteur d'Intelligence Artificielle (IA)

Développement d'un script Python utilisant la vision par ordinateur pour la reconnaissance faciale. Optimisation de la détection via MediaPipe et mise en place d'un algorithme de tracking (CSRT) pour maintenir une fluidité de traitement malgré les ressources matérielles limitées.

6

Interconnexion des systèmes et contrôle d'accès en temps réel

Développement d'un "pont" logiciel permettant au script Python d'interroger la base de données MySQL en temps réel. Le système compare l'identité détectée avec le planning actuel pour autoriser ou refuser l'accès de manière automatisée.

7

Intégration de flux vidéo réseau (RTSP)

Configuration et test de caméras de surveillance distantes. Gestion des protocoles de streaming réseau et résolution des problématiques de latence et de compatibilité des codecs vidéo pour une surveillance déportée efficace.

8

Développement du système d'alerte et de journalisation

Mise en place d'une procédure automatique en cas d'intrusion : capture de preuve photographique, envoi immédiat d'un email d'alerte via protocole SMTP, et archivage dans un journal de bord web sécurisé pour consultation ultérieure.

9

Documentation du projet

Rédaction d’une documentation détaillée décrivant les fonctionnalités, l'architecture et les spécifications techniques du projet et les difficultés rencontrées.

Outils utilisés

WAMP PHP MySQL / PhpMyAdmin HTML5 / CSS3 JavaScript Visual Studio Code Navigateur Web Python RTSP

Front-end

- Création d’interfaces utilisateur ergonomiques

Base de données

- Conception d’un modèle relationnel

- Gestion des relations entre tables

Gestion et configuration de flux vidéo IP (RTSP)

- Paramétrage des URL de flux pour l'interfaçage avec le logiciel de surveillance.

- Analyse et résolution des problématiques de bande passante et de latence réseau impactant la fluidité de la reconnaissance faciale.
