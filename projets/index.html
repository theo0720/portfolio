Projet scolaire Réseau · 2024

Mise en place d'une infrastructure réseau pour une organisation

1 Contexte

Le projet scolaire s'inscrit dans le cadre de la Maison des Ligues de Lorraine (M2L), une organisation fictive, simulant un environnement professionnel.

Objectif : Concevoir et mettre en place une infrastructure réseau répondant aux besoins des ligues sportives hébérgées.

2 Problématique

Comment concevoir et déployer une infrastructure réseau fonctionnelle permettant la communications entre les équipements, la gestion des utilisateurs et la mise à disposition de services réseau (DHCP, DNS, Active Directory) ?

3 Réalisation

  • Conception du schéma réseau et plan d'adressage IP (RFC 1918)
  • Configuration des VLANs et trunks sur commutateur Cisco
  • Déploiement d'un serveur DHCP par VLAN avec les plages d'adresses appropriées
  • Installation et configuration d'un serveur DNS et Active Directory
  • Création et gestion des utilisateurs et machines
  • Tests de bon fonctionnement (connectivité, accès, services)
  • Rédaction d'une documentation technique complète (schémas, configs, tests)

4 Outils utilisés

Machines virtuelles Outils réseau Windows Server Debian Linux Diagrams.net

5 Résultat

Infrastructure opérationnelle : VLANs segmentés, DHCP et DNS fonctionnels, utilisateurs pouvant accéder aux ressources. Tous les tests de connectivité et de blocage sont concluants.

6 Compétences mobilisées

Configuration VLAN Routage IP Configuration DNS DHCP Server Documentation technique Adressage IPv4 Gérer les utilisateurs dans un domaine Active Directory

Projet scolaire Réseau · 2025

Déploiement d'une infrastructure réseau simulée

1 Contexte

Projet scolaire portant sur le contexte CUB, une entreprise fictive spécialisée dans la mise à disposition d’espaces de coworking via une plateforme numérique.

CUB propose à des entreprises de louer des salles (réunion, formation…) et s’appuie sur une infrastructure informatique pour gérer les accès aux services, les ressources réseau et les outils collaboratifs.

Objectif : Se servir de ce contexte comme support pédagogique pour concevoir et déployer une infrastructure réseau simulée.

2 Problématique

Comment concevoir et mettre en place une infrastructure réseau sécurisée et organisée pour répondre aux besoins de l’entreprise CUB, notamment la segmentation du réseau (VLAN), la gestion des utilisateurs, la mise en place de services (DHCP, DNS, Active Directory) et l'accès aux ressources internes et à Internet ?

3 Réalisation

  • Conception d’une architecture réseau adaptée à l’entreprise
  • Mise en place de VLANs pour segmenter le réseau
  • Configuration des équipements (switch, routeur, pare-feu)
  • Mise en place des services (DHCP, DNS, Active Directory)
  • Installation d’un serveur (Linux/Debian)
  • Tests de connectivité et validation du fonctionnement
  • Rédaction de la procédure de déploiement et de maintenance

4 Outils utilisés

Switch / Routeur Windows Server Machine Linux (Debian) Commandes réseaux Environnement virtualisé

5 Résultat

Une infrastructure réseau fonctionnelle et segmentée permettant la mise en place d'une communication entre les différents VLANs, de services réseau opérationnels, la gestion des utilisateurs via Active Directory et l'accès aux ressources et à Internet fonctionnel. Reproduction d'une architecture réseau réaliste d’entreprise.

6 Compétences mobilisées

DHCP server Pare-feu (Stormshield) DNS server Serveur Web VLAN Active Directory Redondance de serveur

Projet scolaire Infrastructure · 2026

Aux Claviers Citoyens — Déploiement d'une infrastructure e-sport en environnement militaire

1 Contexte

Projet initié par le Commissariat au Numérique de Défense (CND), en partenariat avec des lycées BTS SIO et CIEL. L'objectif est de mettre en œuvre les services numériques nécessaires à la pratique du e-sport en environnement militaire, en déployant une infrastructure réseau et système complète capable de supporter une journée de compétition.

Objectif SISR : Déployer les postes joueurs, les moyens réseaux et les serveurs hébergeant l'application de gestion du tournoi développée par les étudiants SLAM.

2 Problématique

Comment concevoir et déployer une architecture réseau et système complète permettant d'héberger une application web, d'interconnecter des postes clients et des serveurs via une passerelle, tout en garantissant l'accès à Internet et la possibilité de déployer des logiciels à distance sur les postes joueurs ?

3 Réalisation

  • Conception de l'architecture réseau : plan d'adressage IP (sous-réseau /20 redécoupé), schéma d'infrastructure
  • Installation et configuration d'une passerelle assurant l'interconnexion entre Internet, le sous-réseau postes clients et le sous-réseau serveurs
  • Déploiement d'un serveur d'application Linux (sans interface graphique) avec une pile LAMP — Apache2, PHP 8.2, MariaDB et PhpMyAdmin
  • Configuration des postes clients sur un sous-réseau dédié avec adressage fixe (bonus : DHCP)
  • Déploiement de l'application de gestion du tournoi depuis GitHub sur le serveur LAMP
  • Installation d'un serveur de déploiement à distance de logiciels sur les postes clients
  • Tests de connectivité, d'accès à l'application et de déploiement du navigateur Firefox 146 sur les postes clients
  • Rédaction des livrables : plan d'adressage, schéma réseau, documentation de déploiement

4 Outils utilisés

VirtualBox Linux (sans GUI) Apache2 PHP 8.2 MariaDB PhpMyAdmin GitHub Diagrams.net

5 Résultat

Infrastructure opérationnelle le jour de la compétition sur la Base aérienne 721 de Rochefort. Les postes clients accèdent à Internet et à l'application de gestion du tournoi. Firefox 146 déployé à distance sur les postes joueurs. L'application de scoring fonctionne en réseau et le serveur LAMP répond correctement aux requêtes des clients.

6 Compétences mobilisées

Architecture réseau Plan d'adressage IP Configuration passerelle Serveur LAMP Virtualisation Déploiement à distance Documentation technique Travail en équipe
